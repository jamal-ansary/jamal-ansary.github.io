Grounding Is a Reasoning Problem

Grounding is more than finding the correct pixel. It requires understanding intent, context, semantics, and planning the next action.

Most multimodal agents treat grounding as a localization problem: finding the correct button, icon, or object on the screen. While localization is an important component, it is only one piece of what modern multimodal agents must actually solve.

Consider the instruction "Go to the Settings app." If the agent is currently inside the Navigation app, there is no Settings icon to click. A purely visual grounding system cannot solve this task because the target does not exist on the current screen.

Instead, the agent must recognize its current state, infer that the Settings application is likely accessible from the Home screen, navigate Home, and only then locate the Settings app. The correct action is therefore not simply localization—it is reasoning about what action should be taken before localization even becomes possible.

This note argues that grounding is fundamentally a reasoning problem involving user intent, interface semantics, application state, and sequential decision making, rather than only a perception problem.
